## 내용 정리
### 소개
- Spring Batch는 아래와 같은 작업에 필요하다고 한다.
  - 주기적으로 사용자와 상호 작용이 없이 처리되는 대량의 작업
  - 내/외부에서 유입되는 대량 데이터의 formating/validation/processing이 필요한 작업


- Spring Batch는 `lightweight`, `comprehensive` 배치 프레임 워크이다.
